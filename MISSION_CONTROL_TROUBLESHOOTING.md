# 🔧 Mission Control - Guía de Troubleshooting\n\n## Problemas Comunes y Soluciones\n\n### ❌ Problema: Panel no carga / Error 404\n\n**Síntomas:**\n- URL devuelve HTTP 404\n- Página en blanco\n- Error \"Cannot GET /mc.html\"\n\n**Soluciones:**\n\n1. Verifica que sinkia-api está corriendo:\n```bash\npm2 list | grep sinkia-api\n```\nDebe mostrar status \"online\"\n\n2. Verifica el puerto 3001:\n```bash\nlsof -i :3001\n```\nDebe mostrar Node.js escuchando en puerto 3001\n\n3. Reinicia el servidor:\n```bash\npm2 restart sinkia-api\n```\n\n4. Verifica los logs:\n```bash\npm2 logs sinkia-api --lines 50\n```\n\n---\n\n### ❌ Problema: Panel carga pero datos no aparecen\n\n**Síntomas:**\n- Panel carga (HTML visible) pero métricas en blanco\n- Contentadores \"Cargando...\"\n- Sin datos de RAM/Disco\n\n**Soluciones:**\n\n1. Verifica el token de autenticación en el navegador (F12 → Console):\n```javascript\n// Debería devolver datos\ncurl('http://localhost:3001/api/admin/system', {\n  headers: { 'x-admin-token': 'sinkia2026' }\n})\n```\n\n2. Verifica que el token es correcto (debe ser: `sinkia2026`)\n\n3. Comprueba los logs de sinkia-api:\n```bash\npm2 logs sinkia-api\n```\nBusca errores de autenticación\n\n4. Limpia caché del navegador:\n- Chrome: Ctrl+Shift+Delete\n- Safari: Cmd+Option+E\n- Firefox: Ctrl+Shift+Delete\n\n---\n\n### ❌ Problema: Botones de acción no funcionan\n\n**Síntomas:**\n- Click en \"Restart\" no hace nada\n- Click en \"Ver Logs\" no abre modal\n- Comandos no se ejecutan\n\n**Soluciones:**\n\n1. Verifica que el token tiene permisos:\n```bash\ncurl -H \"x-admin-token: sinkia2026\" http://localhost:3001/api/admin/docker/test/restart\n```\nDeberá responder (aunque sea error 404 en docker \"test\")\n\n2. Si la respuesta es 403 o 401, el token es incorrecto\n\n3. Verifica que tienes permisos en sinkia-api (reinicia si es necesario)\n\n4. Abre la consola del navegador (F12) para ver errores\n\n---\n\n### ❌ Problema: Docker / PM2 no muestran procesos\n\n**Síntomas:**\n- \"Sin contenedores Docker\"\n- \"Sin procesos PM2\"\n- Listas vacías cuando hay procesos activos\n\n**Soluciones:**\n\n1. Verifica que Docker/PM2 están corriendo:\n```bash\ndocker ps\npm2 list\n```\n\n2. Verifica que sinkia-api tiene acceso a Docker:\n```bash\n# Debería listar contenedores\ndocker ps\n```\n\n3. Reinicia el servicio de admin:\n```bash\npm2 restart sinkia-api\n```\n\n4. Verifica los logs:\n```bash\npm2 logs sinkia-api | grep -i docker\n```\n\n---\n\n### ❌ Problema: Terminal Web no funciona\n\n**Síntomas:**\n- Terminal web en blanco\n- No se escriben comandos\n- Error en consola del navegador\n\n**Soluciones:**\n\n1. Terminal web requiere WebSocket. Verifica conectividad:\n```bash\n# Desde navegador, abre consola (F12)\nws = new WebSocket('ws://localhost:3001')\n```\nNo debería fallar inmediatamente\n\n2. Verifica firewall:\n```bash\nsudo lsof -i :3001 | grep LISTEN\n```\n\n3. Reinicia sinkia-api:\n```bash\npm2 restart sinkia-api\n```\n\n---\n\n### ❌ Problema: Ollama no muestra modelos\n\n**Síntomas:**\n- \"Sin modelos cargados\"\n- Botones de pull no funcionan\n- Modelos no cargan después de pull\n\n**Soluciones:**\n\n1. Verifica que Ollama está corriendo:\n```bash\nollama list\n```\n\n2. Verifica la API de Ollama:\n```bash\ncurl http://localhost:11434/api/tags\n```\nDebe devolver JSON con lista de modelos\n\n3. Si no hay modelos, intenta descargar uno:\n```bash\nollama pull llama3.2:3b\n```\n\n4. Reinicia Ollama:\n```bash\npm2 restart ollama-proxy\n# o\nkillall ollama\nollama serve\n```\n\n---\n\n### ❌ Problema: Red / Tailscale / Cloudflare no responden\n\n**Síntomas:**\n- \"Cargando...\" infinitamente en pestaña Red\n- Tailscale status sin información\n- Cloudflare Tunnel sin estado\n\n**Soluciones:**\n\n1. Verifica Tailscale:\n```bash\ntailscale status\n```\n\n2. Verifica Cloudflare Tunnel:\n```bash\nps aux | grep cloudflare\npm2 list | grep cloudflared\n```\n\n3. Si cloudflared no está corriendo, reinicia:\n```bash\npm2 restart cloudflared-tunnel\n```\n\n4. Verifica logs:\n```bash\npm2 logs cloudflared-tunnel\n```\n\n---\n\n### ❌ Problema: Pipeline IA no responde\n\n**Síntomas:**\n- Tests no ejecutan\n- Modelos no cargan\n- \"Error en test\"\n\n**Soluciones:**\n\n1. Verifica que el servicio IA está corriendo:\n```bash\ncurl http://localhost:3001/api/ai/status\n```\nDeberá devolver JSON con `{\"online\": true}`\n\n2. Si offline, revisa logs:\n```bash\npm2 logs | grep -i \"ai\\|pipeline\\|hermes\"\n```\n\n3. Verifica modelos de Ollama:\n```bash\nollama list\n```\nDeberá haber al menos un modelo cargado\n\n4. Reinicia servicios relacionados:\n```bash\npm2 restart llm-gateway\npm2 restart control-brain\n```\n\n---\n\n## Diagnóstico Completo del Sistema\n\n### Script de diagnóstico automático\n\n```bash\ncat << 'DIAG' > /tmp/mc_diag.sh\n#!/bin/bash\necho \"🔍 Diagnóstico Mission Control\"\necho \"\"\n\necho \"1.
